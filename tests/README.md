# Тестирование UI/UX и E2E (Playwright)

Этот каталог содержит E2E-тесты, тесты адаптивности, здоровья страницы и доступности (a11y), написанные с использованием библиотеки **Playwright** и сканера **Axe-core**.

## Подготовка к запуску тестов

1. **Установка зависимостей и браузеров:**
   В корне проекта выполните установку необходимых пакетов и скачивание браузеров:
   ```bash
   npm install
   npx playwright install chromium
   ```

2. **Настройка локальных переменных окружения:**
   В корне проекта создайте файл `.dev.vars` (он уже добавлен в `.gitignore`):
   ```ini
   SESSION_SECRET=e2e-testing-session-secret-key-32-chars-long-or-more
   ```
   *Примечание:* Для тестов Turnstile-капча и Google OAuth не настраиваются локально, поэтому они будут автоматически отключены бэкендом (что позволяет проводить автоматические E2E-тесты регистрации/входа).

---

## Запуск тестов

### 1. Локальный прогон (рекомендуется)
Для локального запуска выполните команду:
```bash
npm test
```
* **Как это работает:** Playwright автоматически инициализирует чистую локальную базу данных SQLite (`npm run db:init:local`), запустит локальный сервер Cloudflare Wrangler (`npm run dev` на порту `8787`), прогонит все спеки последовательно во избежание блокировок базы данных и завершит сервер.

### 2. Запуск в интерактивном режиме (Playwright UI)
Для отладки и визуального наблюдения за ходом тестов:
```bash
npm run test:ui
```

### 3. Запуск в видимом браузере (Headed)
```bash
npm run test:headed
```

### 4. Запуск против продакшена (или стейджинга)
Для проверки здоровья и доступности живого сайта (без деструктивных изменений БД и E2E-мутаций):
```bash
set BASE_URL=https://sport.ms-cert.workers.dev
npx playwright test tests/page-health.spec.js tests/a11y.spec.js
```
*Примечание:* Так как на проде включена Turnstile капча, E2E тесты мутаций (создание привычек, тренировок) там будут пропущены (`SKIP`).

## Структура тестов

* [helpers.js](helpers.js) — вспомогательные хелперы для быстрой регистрации пользователей по API (минуя ручной ввод логина) и настройки авторизованного контекста браузера.
* [page-health.spec.js](page-health.spec.js) — проверка отсутствия JS-ошибок, битых ассетов (404), наличия мета-тегов, переключения темы (светлая/темная) и языка.
* [auth.spec.js](auth.spec.js) — тестирование UI форм авторизации, валидации полей, регистрации и логаута.
* [navigation-responsive.spec.js](navigation-responsive.spec.js) — проверка адаптивности (сайдбар на десктопе, таб-бар на мобилках) и переключения SPA вкладок.
* [habits.spec.js](habits.spec.js) — E2E сценарий управления привычками (создание, логирование, просмотр деталей логов, изменение и удаление).
* [gym.spec.js](gym.spec.js) — E2E сценарий тренировок (выбор упражнений, добавление подходов, завершение тренировки и проверка графиков статистики).
* [a11y.spec.js](a11y.spec.js) — аудит доступности (axe-core) и проверка поддержки клавиатуры (Tab, Escape).
* [TEST-CHECKLIST.md](TEST-CHECKLIST.md) — интерактивный чеклист с P0/P1 сценариями и селекторами для ручного или полуавтоматического прогона.

## Отчёты о результатах

После выполнения тестов вы можете открыть подробный HTML-отчёт с шагами, логами и скриншотами упавших тестов:
```bash
npm run test:report
```
* Отчёт сохраняется в каталог `playwright-report/`.
* Скриншоты и видеозаписи падений сохраняются в `test-results/`.
