# E2E и UI/UX тестирование (Playwright)

В проекте настроена система сквозного (End-to-End) тестирования интерфейса и доступности на базе Playwright и `@axe-core/playwright`.

Тесты полностью автоматизированы и проверяют ключевые функциональные области приложения как в десктопной версии (Chromium Desktop 1280x800), так и в мобильной эмуляции (mobile-chrome, Pixel 7).

## Структура тестов

Все тесты находятся в папке `tests/` и разделены на 6 спецификаций:

1. **`page-health.spec.js`** — здоровье страницы:
   - Проверка отсутствия критических ошибок JS (console.error, uncaught exceptions).
   - Проверка кодов ответа статических ресурсов и API-конфигурации (200 OK).
   - Валидация мета-тегов в `<head>` (title, viewport, manifest, icon).
   - Корректность переключения тем оформления (светлая/темная) с сохранением в `localStorage`.
   - Локализация приложения (переключение RU/EN) и проверка `data-i18n`.

2. **`auth.spec.js`** — аутентификация:
   - Доступность экрана `#auth` для неавторизованных пользователей.
   - Переключение вкладок Вход/Регистрация.
   - Ошибки валидации (неверный пароль, слишком короткий пароль).
   - Успешный цикл регистрации нового пользователя и корректный выход из системы (Sign Out).

3. **`navigation-responsive.spec.js`** — адаптивность и навигация:
   - Проверка скрытия сайдбара на мобильных устройствах и нижнего таб-бара на десктопах.
   - Успешный переход по всем 4 вкладкам приложения (`today`, `calendar`, `stats`, `profile`).
   - Открытие и закрытие модальных окон привычек и Gym Mode (по кнопкам и клику по оверлею).
   - Проверка отсутствия горизонтального скролла на страницах (`scrollWidth <= clientWidth`).

4. **`habits.spec.js`** — E2E привычек:
   - Создание числовых и простых (Simple) привычек.
   - Быстрое логирование (+1) и логирование через модалку.
   - Переход в детальный просмотр с графиками, статистикой и ачивками.
   - Редактирование и удаление записей логов.
   - Изменение и удаление самих привычек.

5. **`gym.spec.js`** — E2E Gym Mode:
   - Проверка инициализации базового набора упражнений.
   - Добавление подходов (вес, повторения, калории).
   - Завершение тренировки и проверка сохранения данных.
   - Отображение графиков, Muscle Heatmap и сводной статистики в табе `stats`.

6. **`a11y.spec.js`** — доступность (WCAG):
   - Прогон аудита `axe-core` на основных экранах (экран входа, дашборд, профиль, открытые модалки).
   - Проверка семантики полей, наличия ярлыков (labels) и доступных имен у кнопок-иконок.
   - Навигация с клавиатуры (Tab-фокус) и закрытие диалоговых окон по нажатию клавиши `Escape`.

## Запуск тестов

Тестовое окружение полностью изолировано и запускает локальный dev-сервер wrangler с чистой базой данных SQLite D1.

### Команды запуска
```bash
# Установка зависимостей и браузеров
npm install
npx playwright install chromium

# Запуск всех 40 тестов (в headless-режиме)
npm test

# Запуск тестов в визуальном интерактивном интерфейсе UI
npm run test:ui

# Запуск тестов в режиме отображения браузера (headed)
npm run test:headed

# Просмотр последнего сгенерированного HTML-отчета
npm run test:report
```

### Тестирование против Production
Для проверки работоспособности на прод-окружении (только read-only спеки, так как мутации блокируются капчей Turnstile):
```bash
set BASE_URL=https://trecker.ms-cert.workers.dev
npx playwright test tests/page-health.spec.js tests/a11y.spec.js
```
